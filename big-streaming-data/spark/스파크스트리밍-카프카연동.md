## RDD로 프로그래밍하기
1. 기초
- 외부 데이터에서 입력 RDD(분산되어 있는 변경 불가능한 객체 모음) 를 만든다.
- filter() 같은 트랜스 포메이션은 새로운 RDD를 정의한다.
- 재사용을 위한 중간 단계의 RDD 보존을 위해 스파크에 persist() 요청
- 스파크가 최적화한 병렬 연산 수행을 위해 액션을 시작함.
- Read-Only

2. 연산
- 트랜스포메이션 -> 리니지를 기록해둠
- 액션 -> 실제 최종 결과값을 되돌려 주거나 외부 저장소에 값을 기록 하는 작업
- lazy-execution

3. 캐싱
- persist()
